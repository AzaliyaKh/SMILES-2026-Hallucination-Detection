# Hallucination Detection Solution

## Repository submission requirements

The GitHub repository should contain:

- the implemented solution files:
  - `aggregation.py`
  - `probe.py`
  - `splitting.py`
- `results.json` with the evaluation results;
- `predictions.csv` generated by `solution.py`;
- this report file: `SOLUTION.md`.

The solution should be reproducible by running:

```bash
pip install -r requirements.txt
python solution.py
```

The main script `solution.py` extracts hidden states from the Qwen model, trains the hallucination probe, evaluates the model, saves `results.json`, and generates predictions for the hidden test set.

---

## Task overview

The task is to detect hallucinations in model responses. Each example consists of a `prompt`, a `response`, and a binary label:

- `0` — truthful answer;
- `1` — hallucinated answer.

The project uses hidden states from the language model `Qwen/Qwen2.5-0.5B`. These hidden states are converted into feature vectors in `aggregation.py`, and then a binary classifier from `probe.py` predicts whether the answer is truthful or hallucinated.

The main pipeline is:

```text
prompt + response
        ↓
Qwen/Qwen2.5-0.5B
        ↓
hidden states
        ↓
aggregation.py
        ↓
feature vector
        ↓
probe.py
        ↓
0 / 1 prediction
```

## Changes in `probe.py`

### 1. Adding Dropout

To reduce overfitting, I added a `Dropout` layer to the probe classifier. Dropout randomly disables part of the neurons during training, which makes the model less likely to memorize the training set.

The original architecture was:

```python
self._net = nn.Sequential(
    nn.Linear(input_dim, 256),
    nn.ReLU(),
    nn.Linear(256, 1),
)
```

After adding Dropout:

```python
self._net = nn.Sequential(
    nn.Linear(input_dim, 256),
    nn.ReLU(),
    nn.Dropout(0.3),
    nn.Linear(256, 1),
)
```

This improved the test performance:

```text
Test Accuracy: 74.04% → 75.96%
Test F1:       82.12% → 84.08%
```

At the same time, train accuracy decreased, which means the model became less overfitted.

### 2. Adding weight decay

I also tried adding `weight_decay` to the Adam optimizer. The idea was to penalize large weights and reduce memorization of the training data.

However, this did not improve the model. Although train accuracy decreased, the test metrics also became worse:

```text
Test Accuracy: 75.96% → 74.04%
Test F1:       84.08% → 83.44%
```

A weaker value, `weight_decay=1e-5`, also worsened the results. Therefore, weight decay was not used in the final version.

### 3. Reducing the number of epochs

Since the dataset contains only 689 examples, I tested reducing the number of training epochs from 200 to 100 and 150.

This did not improve the result. With fewer epochs, the model most likely became undertrained: both validation and test metrics decreased.

Therefore, the final version keeps:

```python
for _ in range(200):
```

### 4. Adding an additional hidden layer

I also tested a more complex probe architecture with an additional hidden layer:

```python
self._net = nn.Sequential(
    nn.Linear(input_dim, 256),
    nn.ReLU(),
    nn.Dropout(0.3),

    nn.Linear(256, 64),
    nn.ReLU(),
    nn.Dropout(0.2),

    nn.Linear(64, 1),
)
```

This made the model more complex, but the test performance became worse. The model started fitting the training data better, while generalization to the test split decreased.

For this small dataset, the two-layer architecture was too complex, so it was not used in the final solution.

## Changes in `aggregation.py`

The baseline aggregation strategy used:

```text
last layer + last real token
```

This is simple and compact, but it can potentially lose information because the whole `prompt + response` sequence is represented by only one token vector.

Possible problems with this approach:

1. The whole text is compressed into a single vector.
2. A hallucination can appear in the middle of the response.
3. The last token can be punctuation or a special token.
4. The last layer is not always the best layer for classification.
5. Useful information can be distributed across different tokens and layers.

Because of this, I tested several alternative aggregation strategies.

### 1. Mean pooling over the last layer

I tried replacing the last-token representation with mean pooling over all real tokens from the final layer.

This approach performed poorly. The model almost returned to the majority-class baseline and did not separate hallucinated and truthful answers well. AUROC was close to 50%, which indicates almost random class separation.

Therefore, mean pooling over all tokens was not used.

### 2. Last token + mean pooling

I also tested combining the last token representation with mean pooling from the last layer.

This increased the feature dimension from 896 to 1792, but the result was worse. The additional mean pooling features likely diluted the useful signal from the last token.

This option was also rejected.

### 3. Last token from the last 4 layers

Another experiment used the last real token from the last 4 layers:

```text
4 layers × 896 = 3584 features
```

This increased the amount of information given to the classifier, but did not improve the test metrics. The model became more prone to overfitting.

### 4. Last token from the last 2 layers

I also tried using the last real token from the last 2 layers:

```text
2 layers × 896 = 1792 features
```

This was less aggressive than using 4 layers, but it still did not improve the baseline aggregation.

### 5. Average of the last 2 layers

Instead of concatenating the last 2 layers, I tried averaging them to keep the feature dimension equal to 896.

This also did not improve the result. The useful signal seemed to be stronger in the final layer alone.

### 6. Weighted average of the last 2 layers

I also tested a weighted average:

```text
0.3 * previous layer + 0.7 * last layer
```

This gave more importance to the last layer, but the result was still worse than using only the last layer.

## Final selected approach

The experiments showed that increasing the complexity of aggregation usually made the result worse. Therefore, the final solution keeps the compact baseline aggregation:

```python
def aggregate(
    hidden_states: torch.Tensor,
    attention_mask: torch.Tensor,
) -> torch.Tensor:
    """
    Aggregate hidden states into a single feature vector.

    Default baseline: last real token from the final layer.
    """

    layer = hidden_states[-1]

    real_positions = attention_mask.nonzero(as_tuple=False).squeeze(-1)
    last_pos = int(real_positions[-1].item())

    feature = layer[last_pos]

    return feature
```

The final probe classifier uses one hidden layer and Dropout:

```python
self._net = nn.Sequential(
    nn.Linear(input_dim, 256),
    nn.ReLU(),
    nn.Dropout(0.45),
    nn.Linear(256, 1),
)
```

The optimizer remains Adam without weight decay:

```python
optimizer = torch.optim.Adam(self.parameters(), lr=1e-3)
```

The number of epochs remains:

```python
for _ in range(200):
```

## Conclusion

The final solution uses Qwen hidden states as fixed features and trains a lightweight binary classifier on top of them. The best-performing approach keeps the aggregation simple and adds regularization to the classifier through Dropout.

The final model improves over the initial baseline and reduces overfitting while maintaining a simple and reproducible architecture.
